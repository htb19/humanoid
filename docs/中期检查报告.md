# 人形机器人项目中期检查报告

## 一、项目概述

本项目旨在开发一套完整的人形机器人上半身操作系统，涵盖从硬件驱动、运动控制、仿真环境到基于强化学习的自主抓取策略训练的全链路。机器人上半身具备双臂（每臂6自由度）、颈部（2自由度）、双手夹爪及头部/腕部相机，总计16个主动关节。

项目采用 ROS 2 Humble 作为核心通信与控制框架，结合 MoveIt 2 进行运动规划，Gazebo（Ignition Fortress）进行物理仿真，并独立搭建了基于 Isaac Sim 和 MuJoCo 的强化学习训练环境。

### 2.2 ROS 2 功能包说明

| 功能包 | 功能 |
|---|---|
| robot_description | 机器人 URDF 模型、mesh 文件、ros2_control 配置 |
| robot_commander | 任务调度节点，订阅关节/位姿/夹爪话题，调用 MoveIt 执行 |
| robot_moveit_config | MoveIt 2 运动规划配置（规划组、运动学求解器、碰撞矩阵等） |
| robot_hardware | 硬件接口，通过 CAN-FD 分析仪驱动实际电机 |
| robot_simulation | Gazebo 仿真启动文件与世界配置 |
| servo_control | 基于 MoveIt Servo 的实时伺服控制 |
| robot_keyboard_control | 键盘遥操作（关节模式 + 笛卡尔模式） |
| camera_calibration | 基于 ArUco 的手眼标定 |
| robot_interfaces | 自定义消息类型（PoseCommand 等） |
| robot_rl_training | ROS 2 集成的 RL 训练封装 |
| gz_ros2_control | Gazebo-ros2_control 桥接（源码编译） |

## 三、已完成工作

### 3.1 机器人建模与描述

- 完成 SolidWorks 导出的 URDF 模型集成，包含完整的惯性参数、碰撞几何和视觉 mesh
- 使用 xacro 宏实现模块化配置：支持实机模式与 Gazebo 仿真模式切换
- 配置相机光学坐标系（camera_optical_link.xacro）
- 完成 ros2_control 硬件接口配置，支持位置控制模式

### 3.2 运动控制系统

- 搭建基于 ros2_control 的控制框架，配置 5 组 JointTrajectoryController（左臂、右臂、左夹爪、右夹爪、颈部）
- 集成 MoveIt 2 运动规划，支持关节空间和笛卡尔空间规划
- 实现 robot_commander 节点，统一管理关节控制、位姿控制和夹爪控制话题
- 实现键盘遥操作功能，支持关节直接控制和笛卡尔末端控制两种模式
- 集成 MoveIt Servo 伺服控制，支持实时速度指令输入

### 3.3 仿真环境

- 配置 Gazebo（Ignition Fortress）仿真环境，实现机器人物理仿真
- 源码编译 gz_ros2_control 桥接包，实现仿真与 ros2_control 的对接
- 支持仿真模式与实机模式的无缝切换（通过 launch 参数 `use_simulation`）
- 配置 RViz 可视化与 RQT 轨迹控制器调试工具

### 3.4 硬件接口

- 开发 robot_hardware 包，实现基于 CAN-FD 协议的电机驱动接口
- 使用 USBCANFD-200U 分析仪，配置 1Mbps 仲裁段 + 5Mbps 数据段波特率
- 为每个关节配置电机通道和 ID 映射
- 支持实机与仿真（MockSystem）的切换

### 3.5 视觉标定

- 搭建基于 ArUco 标记的手眼标定流程
- 实现数据采集与位姿计算服务（save_handeye_pair / compute_handeye）
- 支持在仿真环境中进行标定测试

### 3.6 强化学习训练环境

#### Isaac Sim 环境

- 搭建完全独立于 ROS 2 的 Isaac Sim 训练环境
- 实现 Gymnasium 标准接口的 HumanoidBrickPickEnv 环境
- 完成 URDF 自动解析与导入流程（自动处理 package:// 路径、关节限位解析）
- 实现 PPO 训练流水线，支持多环境并行训练（SubprocVecEnv）
- 实现训练监控回调（TensorBoard 日志、周期性评估、检查点保存）
- 实现确定性抓取演示脚本（demo_pick_brick.py）
- 新增姿态检查工具：初始姿态验证（check_initial_pose.py）和交互式双臂姿态调试（check_pose.py）

#### MuJoCo 环境

- 从 URDF 自动生成 MuJoCo MJCF 资源文件
- 实现右臂子集的砖块抓取环境（PickBrickEnv）
- 设计多项奖励函数：距离奖励、抓取奖励、抬升奖励、成功奖励、动作惩罚、平滑惩罚
- 完成 PPO 训练与评估流水线
- 提供 Conda 环境自动配置脚本

#### 任务定义

- 场景：固定躯干的机器人 + 桌面 + 砖块
- 观测空间：关节位置、关节速度、夹爪开度、末端执行器位置、砖块位置、相对向量
- 动作空间：6个关节位置增量 + 1个夹爪指令
- 成功条件：抓取砖块并抬升至指定高度，保持稳定若干步

## 四、当前存在的问题

1. 笛卡尔空间位姿控制存在规划失败问题，部分目标位姿无法到达，六轴运动受限
2. 末端执行器位姿参考点为 wrist_yaw_joint 坐标系，而非实际夹爪中心位置，导致控制精度偏差
3. URDF 中关节限位范围过大（±3.14 rad），对 RL 训练初期探索效率有负面影响，需要针对训练任务收紧限位
4. MuJoCo 环境目前仅使用右臂子集，尚未扩展到全身模型
5. RL 训练尚未引入域随机化（Domain Randomization）和课程学习（Curriculum Learning）
6. 当前 RL 策略仅使用低维状态观测，尚未接入视觉观测
